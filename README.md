# Spellbrawl

A top-down wizard arena brawler for the browser. Fight AI wizards in best-of-5 rounds,
drafting one of three random boons between every round (Hades-style), with
Smash Bros-style knockback health: hits build your **Pressure %**, and the higher it
climbs, the further you fly. Get pushed off the arena and you lose a stock.

**Live:** https://spellbrawl.vercel.app  
**Repo:** https://github.com/abponcio/spellbrawl  
**Deploy:** pushes to `main` auto-deploy via Vercel Git

## Controls

| Input       | Action                             |
| ----------- | ---------------------------------- |
| WASD        | Move                               |
| Mouse       | Aim                                |
| Left click  | Attack (hold to auto-fire)         |
| Right click | Shield (hold)                      |
| Space/Shift | Dash                               |

## Features

- Solo modes: Duel, Skirmish, Chaos (1–3 AI wizards)
- **Settings:** AI difficulty (Easy / Normal / Hard)
- **Match stats:** damage dealt/taken, blocks, KOs, distance traveled
- **Play Online (UI):** create/join room codes — PartyKit multiplayer server coming next
- Guest Supabase auth: anonymous sign-in, cloud settings sync, match history + lifetime stats
- **Account screen:** lifetime stats, recent matches, guest badge
- **Play Online:** PartyKit 4-player rooms with 6-char codes, authoritative sim, online draft

## Running locally

```bash
npm install
npm run sfx          # generate WAV sound effects (also runs on build)
npm run dev          # Vite client at http://localhost:5173
npm run dev:party    # PartyKit server at ws://127.0.0.1:1999 (separate terminal)
```

Copy `.env.example` to `.env.local`. For online battles locally, run **both** `dev` and `dev:party`.

## GitHub + Vercel deploy

1. Create a repo on GitHub (e.g. `spellbrawl`)
2. Push this project (already done if you cloned from GitHub):

```bash
git remote add origin https://github.com/abponcio/spellbrawl.git
git push -u origin main
```

3. Link Vercel to GitHub for auto-deploy on push — **connected** (`abponcio/spellbrawl` → production on every `main` push)

4. Enable **Anonymous sign-ins** in [Supabase Auth](https://supabase.com/dashboard/project/nunlfinypmuktgyzfult/auth/providers)

## Environment variables

Copy `.env.example` to `.env.local` for development. Set the same keys in **Vercel → Environment Variables**:

| Key | Purpose |
|-----|---------|
| `VITE_SUPABASE_URL` | Supabase project API URL |
| `VITE_SUPABASE_ANON_KEY` | Supabase publishable/anon key (safe in browser) |
| `VITE_PARTYKIT_HOST` | Cloudflare Workers host (e.g. `spellbrawl-party.<account>.workers.dev`) or `localhost:8787` for local dev |

Run `npm run dev:party` alongside `npm run dev` for local online battles. Deploy the party server with `npm run party:deploy` (requires `CLOUDFLARE_ACCOUNT_ID` + `CLOUDFLARE_API_TOKEN` in your shell) and set `VITE_PARTYKIT_HOST` in Vercel to the `*.workers.dev` URL.

Run the SQL in `supabase/migrations/` after creating a Supabase project. Enable **Anonymous sign-ins** in Auth settings for guest mode.

## Tech

- Vite + TypeScript, Canvas 2D
- `src/engine/` — game loop, input, vectors, particles, synth audio
- `src/game/` — wizards, skills, boons, AI, match state, combat stats
- `src/scenes/` — menu, settings, lobby, draft, arena, results
- `src/net/` — Supabase client, guest auth, party WebSocket client, profile + sync
- `src/game/sim/` — authoritative arena sim (shared with Cloudflare Workers server)
- `party/spellbrawl.ts` — Durable Object room handler (lobby → draft → arena)
- `party/worker.ts` + `party/wrangler.jsonc` — Cloudflare Workers entry + deploy config (see `party/package.json`)
- `public/sfx/` — curated WAV sound effects (generated by `npm run sfx`)
- `supabase/migrations/` — profiles, settings, match history, player stats
